# Plan 01-03: Decouple Engine & Verify Codebase Parities

Migrate all gameplay strategy signatures and the game loop Engine to use the new platform-agnostic `Renderer` interface (per decision D-01). Enforce compile typechecks, unit test correctness, and execute smoke testing to ensure complete visual parity.

## Objective

Connect `Canvas2DRenderer` and `Engine` in the web client and complete the refactoring of all strategy drawing signatures to compile and run successfully.

<threat_model>
- **Elevation of Privilege (E)**: Unsanitized custom configuration maps executed inside `DynamicPromptStrategy` causing shell execution or access to cookies/storage in the browser container.
  - *Mitigation*: Ensure no dynamic text parsing leverages `eval()`, `Function()`, or unsafe JSON parsing. Exclude system commands from configuration parsing.
- **Tampering (T)**: Mocking canvas elements or boundaries in tests to inject false-positive success parameters.
  - *Mitigation*: Enforce strict mock assertions in Vitest tests. Validate `InputBoundsProvider` bounds return format.
</threat_model>
