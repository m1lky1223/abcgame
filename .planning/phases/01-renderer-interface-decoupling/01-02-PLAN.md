# Plan 01-02: Migrate Entity Classes & Delegate to Renderer

Extract all path, curve, bezier, and shape drawing logic out of entity classes (`src/characters/draw.ts`, `src/game/Background.ts`, `src/game/OddbodChaser.ts`, `src/game/ZombieChaser.ts`) and place them directly into the platform-specific `Canvas2DRenderer` class (per decision D-02). Refactor the entities to accept a `Renderer` parameter and delegate their rendering (per decision D-01).

## Objective

Move graphics drawing implementation into the renderer layer, freeing the game simulation entities from direct canvas references.

<threat_model>
- **Denial of Service (D)**: High entity/particle counts causing rendering performance degradation or browser freeze (thread blocking).
  - *Mitigation*: Background cloud/sparkle counts are bounded by initialization limits. Character scaling is clamped, and render routines avoid redundant canvas state saves/restores.
- **Tampering (T)**: Altering raw coordinates to trigger canvas memory exceptions or stack errors.
  - *Mitigation*: Implement range guards for position inputs before rendering. Ensure translation math relies on verified state values.
</threat_model>
