# Plan 01-01: Define Renderer Interface & Canvas2DRenderer

Define the platform-agnostic `Renderer` interface and implement the `Canvas2DRenderer` concrete class for HTML5 Canvas 2D. Stub out high-level domain wrappers to compile without circular imports.

## Objective

Create the interface abstraction layer (`Renderer`) and its web target implementation (`Canvas2DRenderer`), validated by a new unit test suite.

<threat_model>
- **Tampering (T)**: Exploitation of renderer text operations (`fillText`/`strokeText`) to inject malicious HTML/JS payload (XSS).
  - *Mitigation*: Canvas 2D's native text APIs do not parse or evaluate HTML markup; they render strings as raw text vectors. No dynamic evaluation or DOM-injection wrappers will be included in the renderer.
- **Denial of Service (D)**: Passing invalid, infinite (NaN/Infinity), or extremely large parameters (e.g., radius, shadow blur, path segments) that lock the browser thread or crash the Canvas context.
  - *Mitigation*: Enforce parameter validation constraints on coordinates. Standard math limits are respected, and recursion is avoided.
</threat_model>
